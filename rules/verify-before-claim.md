# Verify Before Claim Rule

> 回答关于代码状态的问题前，先验证

---

## 触发条件

当用户问以下类型问题时：
- "这个函数做什么的？"
- "当前配置是什么？"
- "这个文件里有xxx吗？"
- "参数值是多少？"

---

## 规则

### 1. 先读取，再回答

```
❌ 错误：凭记忆回答
用户: 这个函数做什么的？
AI: 这个函数用于计算xxx（可能过时/错误）

✅ 正确：先验证
用户: 这个函数做什么的？
AI: 让我先读取这个文件...
[Read file]
这个函数 at src/module.py:45 用于...
```

### 2. 引用来源

回答时标注具体位置：
```
✅ 这个函数 at backtest/runner.py:120 用于执行向量化回测
✅ 当前配置在 config/leaders.yaml 第15行，参数值为...
```

### 3. 不确定时声明

```
✅ 根据上次会话的记忆，应该是xxx，让我验证一下
✅ 我需要先读取文件确认
```

---

## 常见错误场景

| 场景 | 错误做法 | 正确做法 |
|------|----------|----------|
| 函数功能 | 凭记忆描述 | Read 后引用行号 |
| 配置值 | 猜测当前值 | Read 配置文件确认 |
| 参数范围 | 说"大概是" | 打印实际范围 |
| 文件内容 | 假设存在 | Glob/Read 验证 |

---

## 示例对话

```
用户: 当前 Leaders 策略的持仓数量是多少？

❌ 错误回答:
AI: 应该是 10-20 只吧

✅ 正确回答:
AI: 让我确认一下配置...
[Read config/leaders.yaml]
当前配置 at config/leaders.yaml:25 是 `position_count: [10, 20]`
```

---

## 根因分析

从对话历史中发现：
- 用户纠正了 8 次"事实错误"
- 常见原因：AI 凭记忆回答，未验证当前状态
