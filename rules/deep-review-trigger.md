# Deep Review 自动触发规则

> 在合适时机智能建议执行 `/deep-review`

---

## 触发条件

以下场景在代码变更后**自动建议**执行 `/deep-review`：

### 1. 高风险文件（安全相关）

| 文件模式 | 说明 |
|---------|------|
| `**/auth*` | 认证模块 |
| `**/security*` | 安全模块 |
| `**/password*` | 密码处理 |
| `**/token*` | Token 管理 |
| `**/api_key*` | API Key 相关 |
| `**/payment*` | 支付模块 |
| `**/permission*` | 权限控制 |
| `**/user*` | 用户数据 |

### 2. 大规模变更

| 条件 | 阈值 |
|------|------|
| 修改文件数 | ≥ 5 个 |
| 变更行数 | ≥ 200 行 |
| 新增文件 | ≥ 3 个 |

### 3. 安全敏感操作

| 操作 | 说明 |
|------|------|
| 用户输入处理 | 表单验证、请求解析 |
| API 端点 | 新增/修改 REST/GraphQL 端点 |
| 数据库操作 | SQL 查询、ORM 修改 |
| 加密/解密 | 加密算法使用 |
| 文件操作 | 文件读写、上传下载 |
| 外部 API | 调用第三方服务 |

---

## 执行方式

### 轻量提示（推荐）

变更完成后输出：

```
📊 检测到安全相关变更：
   - auth/login.py
   - api/user.py

💡 建议执行: /deep-review
```

**不强制阻塞**，由用户决定是否执行。

### 自动触发（可选）

在 CLAUDE.md 中配置：

```markdown
## 自动审查规则

修改以下文件后自动执行 /deep-review：
- auth/**
- security/**
- payment/**
```

---

## 与 review-trigger.py 的协作

| 工具 | 触发点 | 用途 |
|------|--------|------|
| **review-trigger.py** | git commit | 正则扫描，轻量提示 |
| **本 rule** | 代码变更后 | 智能建议，场景感知 |
| **手动 /deep-review** | 用户需求 | 完整审查 |

**分工**：
- Hook 捕获 commit 时的硬编码问题
- Rule 感知变更上下文，智能建议
- 用户保留最终决定权

---

## 实现机制

### Claude Code Rule 集成

规则会在以下时机检查：

1. **Edit/Write 工具调用后**
   - 检查文件路径是否匹配高风险模式
   - 检查变更规模

2. **代码变更完成后**
   - 统计本次会话的修改量
   - 评估是否需要建议审查

3. **任务结束前**
   - 汇总变更摘要
   - 输出建议

---

## 配置示例

用户可在项目 `.claude/rules/` 中自定义：

```markdown
# 项目特定审查规则

## 强制审查
修改以下文件必须执行 /deep-review：
- src/core/crypto.py
- src/api/auth.py

## 跳过审查
以下文件变更不需要触发：
- tests/**
- docs/**
- *.md
```

---

## 最佳实践

1. **不阻塞开发流**：仅提示，不强制
2. **场景感知**：理解变更上下文，减少噪音
3. **渐进式**：从轻量扫描到深度审查
4. **团队协作**：可按项目定制规则
