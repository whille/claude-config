# Markdown 智能标注（LLM 驱动版）

从读者价值角度分析文章，使用 LLM 总结洞察，自动标注最有帮助的内容。无需硬编码关键词，适用于任何类型的文档。

## 核心优势（vs 旧版规则匹配）

| 维度 | 旧版（规则匹配） | 新版（LLM 驱动） |
|------|------------------|------------------|
| **准确性** | 低（依赖规则覆盖） | 高（语义理解） |
| **泛化性** | 差（需手动配置） | 强（自动适应） |
| **维护成本** | 高（需更新规则） | 低（自动学习） |
| **成本** | 低（本地执行） | 中（需LLM调用） |

## 核心流程

```
原文 → LLM总结+洞察+关键词 → 定位关键词 → 应用标注 → 验证
```

### 详细流程

1. **分段**：按标题或固定长度分割（每段200-400行）
2. **LLM分析**：每段生成总结、洞察、关键词
3. **去重**：全局去重，每个词只标第一次
4. **标注**：在原文中定位关键词，应用 span 标记
5. **验证**：确保原文内容不变

## 标注理念

**核心问题**：这段内容对读者有什么价值？

不是"这段很重要"，而是"这段能让读者获得什么"。

## 标注类型（从读者收获角度）

| 类型 | 读者价值 | 颜色 | Class |
|------|----------|------|-------|
| **洞见** | 颠覆认知、让人恍然大悟 | 浅黄 | `hl-conclusion` |
| **方法** | 可直接应用的行动指南 | 浅蓝 | `hl-concept` |
| **数据** | 支撑论点的关键证据 | 浅绿 | `hl-data` |
| **警示** | 需要避免的坑、风险提示 | 浅橙 | `hl-warning` |

## 标注原则（严格执行）

1. **读者价值优先**：每次标注前问"这对读者有什么用？"
2. **最少标注**：每段最多 1-2 处，全书控制在段落数的 10%
3. **可行动性**：优先标注能指导行动、改变认知的内容
4. **仅标注核心词**：2-6 字，不标整句
5. **全局去重**：同一个词在全文中只标注第一次出现
6. **年份不标**：年份一般不是重点，除非是关键时间节点

## 判断标准

### 洞见（浅黄 hl-conclusion）

标注标准：能让读者"啊，原来如此"的内容

✅ 标注：
- 颠覆常识的观点
- 揭示事物本质的定义
- 高度概括的原则

❌ 不标：
- 正确但显而易见的陈述
- 作者的主观感受
- 重复强调的内容

### 方法（浅蓝 hl-concept）

标注标准：读者可以直接照着做的内容

✅ 标注：
- 具体可执行的方法
- 明确的操作步骤
- 实用的经验总结
- **个人核心理念**（思想、价值观、原则）

❌ 不标：
- 抽象的建议（"要努力"）
- 无法验证的方法
- 过于复杂的流程

**个人理念类识别规则**：

```
识别"个人核心理念/原则"：

特征：
- 涉及"思想"、"价值观"、"原则"、"理念"
- 是一个人要建立的核心理念
- 对成长有指导意义

示例：
- "自己的思想" → 独立思考能力
- "价值观" → 人生原则
- "自由思考的环境" → 成长环境
```

### 数据（浅绿 hl-data）

标注标准：能增强说服力或提供具体参照的内容

✅ 标注：
- 具体的金额、比例
- 关键时长（如"18个月"）
- 关键节点数据

❌ 不标：
- **年份**（如"1967年"、"2013年"）- 一般不是重点
- 无意义的数字（页码、序号）
- 过于精确但无参考价值的数据

## 去重规则

| 类型 | 规则 |
|------|------|
| **人名** | 全文只标注第一次出现 |
| **概念词** | 全文只标注第一次出现 |
| **数据** | 相似数据只标注最具代表性的 |
| **洞见** | 核心原则可重复标注（最多2-3次） |

### 警示（浅橙 hl-warning）

标注标准：能帮读者避坑的内容

✅ 标注：
- 常见错误
- 风险提示
- 必须遵守的原则
- **民间谚语/俗语**（蕴含人生智慧）

❌ 不标：
- 正面的建议（那是"方法"）
- 已经过时的警告

### 民间智慧/谚语（浅橙 hl-warning）

标注标准：简短有力、易记、有指导意义的民间智慧

**识别规则（泛化提示词）**：

```
民间谚语/俗语识别特征：

1. 结构特征
   - 短句（5-15字）
   - 对仗结构（前后对称）
   - 押韵或朗朗上口
   - 常用比喻、借代

2. 语义特征
   - 包含人生智慧
   - 警示或劝诫
   - 总结规律或道理
   - 具有普遍适用性

3. 语言特征
   - 口语化表达
   - 常带引号（作者引用）
   - 形象生动

4. 内容类型
   - 处世智慧："害人之心不可有，防人之心不可无"
   - 商业规律："买的没有卖的精"
   - 警示格言："想巧就有拙"、"天上不会掉馅饼"
   - 利益驱动："为人不图三分利，没人起个早五更"
```

**识别 Prompt 模板**：

```
分析文本，找出所有民间谚语/俗语/格言：

判断标准：
1. 是否短小精悍（5-15字）
2. 是否朗朗上口（对仗/押韵）
3. 是否蕴含智慧（人生道理/警示）
4. 是否易传播（口语化）

找到后用 hl-warning 标注，仅标注谚语本身（不含引号）。
全文同类谚语仅标注第一次出现。
```

## 输出格式

### 1. 文件头部插入 style 块

```html
<style>
.hl-concept { background-color:#d1ecf1; padding:2px 4px; border-radius:3px; }
.hl-conclusion { background-color:#fff3cd; padding:2px 4px; border-radius:3px; }
.hl-data { background-color:#d4edda; padding:2px 4px; border-radius:3px; }
.hl-warning { background-color:#fff2e6; padding:2px 4px; border-radius:3px; }
</style>
```

### 2. 标注用法

```html
<span class="hl-concept">傅海棠</span>是期货投资传奇人物。
他在<span class="hl-data">18个月</span>内从<span class="hl-data">5万</span>做到<span class="hl-data">1.2亿</span>。
<span class="hl-warning">注意</span>：他的方法需要极致耐心。
<span class="hl-conclusion">供求决定趋势</span>是他的核心理念。
```

## 约束规则

1. **密度控制**：每 4-6 行 1 个重点，单段最多 3 处标注
2. **标注长度**：仅标注 2-8 字的核心词，禁止标注整句
3. **保持原文结构**：不修改、不增删原文内容
4. **禁止原生 MD 标记**：不使用 `==`、`**`、`*` 等
5. **核心优先**：仅标注对理解必不可少的内容
6. **禁止标注套话**：如「研究表明」「我们认为」等不标

## 执行流程（Claude 内部）

### 1. 用户调用

```
/md-highlight <文件路径>
```

### 2. Claude 执行流程

**Step 1: 分段分析**
- Python 脚本分段（每段200-400行）
- 每段独立处理

**Step 2: LLM 分析（Claude 内部调用）**
- 无需外部 API 调用
- Claude 直接分析每段内容
- 生成总结、洞察、关键词

**Step 3: 定位与标注**
- 在原文中定位关键词
- 全局去重（每个词只标第一次）
- 应用 span 标记

**Step 4: 验证**
- 确保原文内容不变
- 检查标注密度（4-25行/处）
- 检查标注长度（2-10字）

**Step 5: 输出**
- 生成 `_highlighted.md` 文件
- 报告标注数量

## 大文件处理

当文件超过 500 行时：
- 按 `##`、`###` 标题分段
- 每段 200-400 行
- 逐段读取 → 标注 → 写回

## 注意事项

1. 标注后文件可在浏览器中查看高亮效果
2. Markdown 编辑器可能不渲染 CSS，建议用浏览器打开
3. 原文件会被覆盖，建议先备份
